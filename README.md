# EasyAEC

EasyAEC 是一个基于 WinForms（C#）+ 本地 AEC 核心（C++ DLL）的实时回声消除工具。

## 当前主要能力

- 三路设备路由：
  - 监听输入（麦克风）
  - 监听输出（参考）
  - 音频输出（处理后）
- AEC 实时处理（10ms 帧）
- 运行态自动延迟补偿标定
- 配置自动保存/加载（`config.json`）
- 最小化到系统托盘（双击恢复）

## 项目结构

- `EasyAEC_GUI/`：WinForms UI 与音频引擎
- `AEC_Core/`：本地 AEC DLL 工程
- `third_party/`：第三方依赖代码

## 构建与运行

### 1) 构建 AEC_Core（x64）

请先在 Visual Studio 中构建 `AEC_Core`（建议 `Release|x64` 或 `Debug|x64`），确保生成 `AEC_Core.dll`。

### 2) 运行 GUI

在项目根目录执行：

```bash
dotnet run --project EasyAEC_GUI/EasyAEC_GUI.csproj -c Debug
```

## 使用说明（简版）

1. 选择三个设备（监听输入、监听输出、音频输出）
2. 点击“开始运行”启动 AEC
3. 点击“自动测算延迟”执行自动标定（会自动测试并回填延迟）
4. 根据需要微调压制强度与延迟补偿

## 配置文件

程序会自动读取/保存以下配置到 `config.json`：

- 设备选择（3 个下拉框）
- 压制强度
- 延迟补偿
- Debug 模式

`config.json` 默认位于程序输出目录（与 exe 同目录）。

## 托盘行为

- 点击窗口右上角 `X`：最小化到托盘（不退出）
- 托盘双击：恢复主窗口
- 托盘菜单“退出”：真正退出程序

## 说明

- 自动标定是“运行态策略”，目标是找到当前链路下可用的延迟补偿值。
- 标定结果会受设备路由、系统音频状态、环境噪声影响。建议在稳定链路下测试。
